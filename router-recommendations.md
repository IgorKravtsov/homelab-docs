# Роутеры с поддержкой WAN + 4G Failover

## Требования

- Ethernet WAN (для проводного интернета)
- 4G LTE SIM слот (как резерв или основной)
- Автоматический failover (при падении провода — переключение на 4G)
- LAN порты для подключения Home Lab по проводу

## Решение проблемы с LAN портами

Все роутеры GL.iNet имеют только 1 LAN порт. Решение — добавить свитч.

### Схема подключения

```
GL.iNet (1 LAN порт)
         │
         ▼ Ethernet
    [Свитч 4-8 портов]
         │
    ┌─────┴─────┬──────┐
    ▼           ▼      ▼
TrueNAS    Proxmox   Другие
```

### DHCP — что это и как работает

**DHCP (Dynamic Host Configuration Protocol)** — протокол автоматического назначения IP-адресов.

#### Как работает

```
Устройство (клиент)                    Роутер (DHCP сервер)
       |                                      |
       |  1. "Мне нужен IP"                  |
       |-------- DHCP Discover (broadcast) ->|
       |                                      |
       |  2. "Могу дать 192.168.1.100"       |
       |<- DHCP Offer ------------------------|
       |                                      |
       |  3. "Ок, давай 192.168.1.100"       |
       |-------- DHCP Request --------------->|
       |                                      |
       |  4. "Подтверждено"                  |
       |<- DHCP ACK --------------------------|
```

#### Что DHCP выдаёт устройству

- **IP адрес** (например 192.168.1.100)
- **Маску подсети** (255.255.255.0)
- **Шлюз** (IP роутера для выхода в интернет)
- **DNS серверы** (доменные имена)

#### В схеме с двумя роутерами

```
Квартирный роутер (DHCP сервер для WAN)
       ↓
GL-iNet получает IP на WAN (192.168.1.x)
       ↓
GL-iNet становится DHCP сервером для своей сети (LAN)
       ↓
TrueNAS и устройства получают IP автоматически
```

### Настройка LAN IP в веб-интерфейсе

**Текущая рабочая конфигурация для этого Home Lab:** оставить GL.iNet на `192.168.8.1/24` и использовать подсеть `192.168.8.x`.

1. Подключи ноутбук к LAN порту GL.iNet
2. Зайди в браузере на `192.168.8.1`
3. Логин/пароль: `root` / `root` или `admin` / `admin`
4. Раздел **Network → LAN** или **LAN Settings**

```
LAN Settings
├─ IP Address: 192.168.8.1    ← текущая рабочая схема
├─ Subnet Mask: 255.255.255.0
└─ DHCP Server: Enabled
       ├─ Start: 192.168.8.100
       └─ End: 192.168.8.254
```

### Частые вопросы

**Нужно ли настраивать квартирный роутер?**

В 95% случаев — нет. Просто подключи GL.iNet к LAN порту квартирного роутера. Квартирный роутер по умолчанию раздаёт DHCP.

**Когда нужно зайти в настройки квартирного роутера:**
- DHCP выключен — включить
- Конфликт подсетей (например, оба роутера в `192.168.8.x`) — изменить подсеть на GL.iNet
- Нужен проброс портов из интернета — настроить проброс на GL.iNet IP

**Как зайти в веб-интерфейс GL.iNet:**
- Подключи ноутбук к LAN порту → в браузере 192.168.8.1
- Или найди IP в списке устройств квартирного роутера
- Или по hostname (например gl-x2000.local)

**Как изменить LAN IP:**
- В веб-интерфейсе: Network → LAN → изменить IP
- После изменения переподключиться по новому адресу
